# NIST Privacy Framework 1.0

**Source:** https://www.nist.gov/privacy-framework
**Released:** January 2020

The NIST Privacy Framework helps organizations identify and manage privacy risks to build
innovative products and services while protecting individuals' privacy.

---

## Core Functions

### IDENTIFY-P
Develop the organizational understanding to manage privacy risk for individuals arising from
data processing.
- Inventory and Map: Data processing ecosystem risk awareness.
- Business Environment: Understand the organization's mission and privacy risk strategy.
- Risk Assessment: Identify privacy risks associated with data processing.

### GOVERN-P
Develop and implement the organizational governance structure to enable ongoing understanding
of the organization's risk management priorities for privacy.
- Policies, Processes, and Procedures: Establish and communicate privacy policies.
- Roles and Responsibilities: Define accountability for privacy risk management.
- Awareness and Training: Train staff on privacy risks and obligations.
- Monitoring and Review: Monitor and improve privacy practices over time.

### CONTROL-P
Develop and implement appropriate activities to enable organizations or individuals to manage
data with sufficient granularity to manage privacy risks.
- Data Processing Policies, Processes, and Procedures: Implement controls aligned to policy.
- Data Processing Management: Manage data collection, use, sharing, and disposal.
- Disassociated Processing: Apply privacy-preserving techniques where possible.

### COMMUNICATE-P
Develop and implement appropriate activities to enable organizations and individuals to have
a reliable understanding and engage in dialogue about how data is processed.
- Communication Policies, Processes, and Procedures: Enable individual awareness of data processing.
- Data Processing Awareness: Notify individuals about data collection and use practices.

### PROTECT-P
Develop and implement appropriate data processing safeguards.
- Data Use Limit: Limit data collection and processing to defined purposes.
- Data Security: Apply security controls to protect personal data.
- Disassociated Processing: Use de-identification, anonymization, and aggregation techniques.
- Data Processing Ecosystem Risk Management: Manage third-party privacy risks.

---

## Key Privacy Principles
- **Data Minimization:** Collect only the data you need for the stated purpose.
- **Purpose Limitation:** Use data only for the purpose it was collected.
- **Storage Limitation:** Retain data only as long as necessary.
- **Accuracy:** Keep personal data accurate and up to date.
- **Transparency:** Be clear about what data is collected and how it is used.
- **Individual Participation:** Allow individuals to access, correct, and delete their data.
- **Security:** Protect personal data against unauthorized access and breaches.
